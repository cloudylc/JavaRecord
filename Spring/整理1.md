# Spring 简介

Spring 最根本的使命：简化 Java 开发

为了降低 Java 开发的复杂性，Spring 采取以下 4种关键策略：

* 基于 POJO 的轻量级和最小侵入性编程；
* 通过依赖注入和面向接口实现松耦合；
* 基于切面和惯例进行声明式编程；
* 通过切面和摸板减少样例式代码；

# 应用上下文

Spring 通过应用上下文（Application Context）装载 bean 的定义并把它们组装起来。Spring 应用上下文全权负责对象的创建和组装。Spring 自带了多种应用上下文的实现，它们之间主要的区别在于如何装载配置。
